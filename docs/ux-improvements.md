# UB Golf — UX Сайжруулалтын Санал

> Бусад golf, community, food ordering app-уудтай харьцуулан UB Golf-д хэрэгжүүлэх боломжтой UX сайжруулалтуудыг тэргүүлэх дарааллаар жагсаасан баримт бичиг.

---

## Харьцуулалтын үндэс

| Ангилал | Харьцуулсан app-ууд |
|---------|---------------------|
| Golf app | GolfNow, Golf Genius, 18Birdies, Golfshot, Golf GameBook |
| Community/Event | Meetup, Strava, Eventbrite, Luma |
| Food ordering | Uber Eats, Toast POS, Square KDS |
| Sports club | TeamSnap, SportsEngine |

---

## 🔴 Яаралтай — Хэрэглэгчийн friction ихтэй

### 1. Navigation тодорхойгүй

**Одоогийн байдал:** Header дээр зөвхөн 2 icon — 👥 Хэрэглэгчид, 🛡️ Admin. Хоолны цэс, Kitchen хаана байдгийг шинэ хэрэглэгч мэддэггүй.

**Харьцуулалт:** Meetup, Strava, Instagram бүгд **bottom navigation bar** ашигладаг. Tab bar-тай app-д хэрэглэгч хамаагүй илүү feature discovery хийдэг нь судалгаагаар батлагдсан.

**Санал — Bottom Navigation Bar:**
```
🏠 Нүүр    ➕ Тоглолт    🍽️ Хоол    👤 Профайл
```
- Хэрэглэгч яаж хоол захиалахаа шууд харна
- Kitchen-д хандах зам тодорхой болно (ажилтны тусдаа нэвтрэлт)

---

### 2. Waiting list байрлал харагдахгүй

**Одоогийн байдал:** Хэрэглэгч waiting list дээр орсноо мэдэх боловч хэдэн дэх байранд байгаа нь харагдахгүй. Хэзээ group руу орохоо мэдэхгүй.

**Харьцуулалт:** Golf Genius, Eventbrite, Luma бүгд waiting list-ийн **байрлалыг** харуулдаг. "You are #2 on the waitlist — 1 person ahead of you" гэх мэт.

**Санал:**
- "Waiting list-ийн **2-р байранд** байна" гэж тодорхой харуулах
- Group рүү орох магадлалыг тодорхойлох (одоогийн бүлгийн хэмжээнээс хамаарч)
- Waiting list-ээс group рүү шилжихэд push notification илгээх

---

### 3. Order tracking хялбаргүй

**Одоогийн байдал:** Хэрэглэгч захиалгаа хаанаас харах вэ? `#/orders/:id` URL мэдэхгүй бол захиалгадаа буцаж очих арга байхгүй. Tracker 2 алхамтай (Төлсөн → Дууссан).

**Харьцуулалт:** Uber Eats, Glovo — 4 алхамт tracker (Захиалсан → Хүлээн авсан → Бэлдэж байна → Бэлэн), шууд хандах товч app-д байнга харагдана.

**Санал:**
- Header эсвэл Профайл дэх "Миний захиалга" shortcut нэмэх
- 4-алхамт tracker: **Төлсөн → Хүлээн авсан → Бэлдэж байна → Бэлэн**
- "Таны захиалга бэлэн боллоо!" push notification

---

## 🟡 Чухал — UX сайжруулалт

### 4. Game card дахь мэдээлэл хязгаарлагдмал

**Одоогийн байдал:** Card дээр цаг, байршил, slot тоо харагдана. Хэрэглэгч карт дарахаас өмнө хэн байгааг, хэдэн slot үлдсэнийг бүтэн харж чаддаггүй.

**Харьцуулалт:** Meetup event card — "12/16 going" progress bar, зохион байгуулагчийн нэр, танил хүн байгаа эсэх. Eventbrite — "3 friends going" гэж харуулна.

**Санал:**
```
┌─────────────────────────────────────┐
│ ⛳ Sky Resort · Бямба 10:00         │
│ ████████████░░░░  12/16 slot        │
│ 👤👤👤 +9 · Та дагадаг 2 хүн байна │
│ [Нэгдэх]                            │
└─────────────────────────────────────┘
```
- Slot progress bar нэмэх
- Тоглогчдын avatar (эхний 3-4) харуулах
- "Таны дагадаг X хүн байна" social proof

---

### 5. Join confirmation modal — шаардлагагүй friction

**Одоогийн байдал:** Description бүхий тоглолтод нэгдэхэд confirm modal гарч, хэрэглэгч заавал уншиж зөвшөөрөх ёстой. Description байхгүй тоглолтод шууд нэгдэнэ.

**Харьцуулалт:** Meetup — "Join" нэг товч дарахад нэгдэнэ, terms нь event page дээрээ харагдана. Eventbrite — confirm хэрэггүй, шууд RSVP.

**Санал:** Description-ийг Game Detail хуудсан дотор байнга харуулж, Join товчийг нэг дарахад болгох. Зайлшгүй шаардлагатай тохиолдолд (нөхцөл заавал зөвшөөрүүлэх шаардлага байвал) хэвэнд нь үлдээнэ.

---

### 6. Food ordering — Game-тай холбоо алдагдана

**Одоогийн байдал:** `#/menu`-ээс орвол `gameId` байхгүй тул захиалга "orphan" болно (ямар тоглолтод зориулсан захиалга вэ гэдэг нь тодорхойгүй). Хоолны цэс нь game context-тэй холбоотой байх ёстой.

**Санал:**
- `#/menu` хуудсаар орохоор идэвхтэй тоглолтуудаасаа сонгох алхам нэмэх
- Game Detail-аас `#/order/:gameId` руу орох нь үргэлжлэх ёстой гол зам байна

---

### 7. Onboarding байхгүй

**Одоогийн байдал:** Шинэ хэрэглэгч нэвтэрмэгц Home feed-д шууд очих боловч юу хийхийг мэддэггүй. Нийгэмлэгийн тухай ойлголт байхгүй.

**Харьцуулалт:** Strava — анхны нэвтрэлтэд sport сонгох, найз хайх онбординг. Meetup — сонирхол сонгох → тохирох event санал болгох.

**Санал — 3 алхамт onboarding (нэвтэрсний дараа нэг удаа харуулах):**
1. "Тавтай морил! Та аль нийгэмлэгт харьяалагдах вэ?" → Community сонгох
2. "Профайлаа бөглөнө үү" → Avatar, нэр
3. "Анхны тоглолтоо олоорой" → Feed рүү шилжих + tutorial tip

## 🟢 Сайжруулалт — Нэмэлт чанар

### 8. Empty state сул

**Одоогийн байдал:** "Тоглолт байхгүй байна" гэсэн текст, CTA товч байхгүй.

**Санал:** Empty state дэх зургийн доор "➕ Тоглолт үүсгэх" товч нэмэх. Жишээ: "Энэ ангилалд тоглолт байхгүй байна. Та өөрөө үүсгэх үү?"

---

### 9. Skeleton loader байхгүй

**Одоогийн байдал:** Feed ачаалахад хоосон хуудас товших хугацаа байна.

**Санал:** Game card-уудын байрлалд skeleton loader (сааралдсан placeholder card) харуулах — Airbnb, Twitter загвараар.

---

### 10. Pull-to-refresh байхгүй

**Одоогийн байдал:** Firebase real-time listener байгаа ч хэрэглэгч дахин ачаалахын тулд хуудсыг бүхэлд нь refresh хийдэг.

**Санал:** Mobile-д pull-to-refresh gesture нэмэх — technical implementation тодорхой байх шаардлагатай (touch event + spinner).

---

### 11. Kitchen "bump" байхгүй

**Одоогийн байдал:** "Дууссан ✓" дарахад дараагийн захиалга автоматаар highlight болдоггүй.

**Харьцуулалт:** Toast KDS, Square KDS — "bump" дарахад дараагийн захиалга автоматаар бүртгэгдэж, ажилтны харц шууд дараагийн захиалга руу очдог.

**Санал:** "Дууссан ✓" дарахад дараагийн идэвхтэй захиалга рүү smooth scroll + brief highlight animation.

---

### 12. Tee time picker visual сайжруулалт

**Одоогийн байдал:** Tee time-уудыг жагсаалтаар харуулдаг.

**Харьцуулалт:** GolfNow — цагийн slot-уудыг **grid calendar** хэлбэрээр харуулна. Хоосон/дүүрсэн slot-ийг нэг харцаар ойлгоно.

**Санал:** Picker-ийг calendar grid болгох — мөр = цаг, багана = 9H/18H.
